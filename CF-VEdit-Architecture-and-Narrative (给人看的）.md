# Part A — 设计架构

## A.0 一句话
输入「源视频 + 编辑指令」,输出「改对该改、锁死该锁」的目标视频:被指令直接波及的区域重渲,因果连带的区域跟着变,其余一切逐帧钉死不动。

## A.1 总览:四块流水线
```
源视频 V + 指令 I
  │
  ├─►【1】Abduction 源反演 (MLLM)
  │      · 读 V + I
  │      · 把 V 反演(inversion)成 renderer 的 VAE latent  ── 记作 源latent
  │      · 源latent = "从源视频能重建的一切" = 不变量先验
  │
  ├─►【2】因果 Planner
  │      解析指令 → do(改什么),输出两类东西(都不是 mask):
  │      · 区域 query 向量(少量、稀疏)= "哪些区块会被波及" 的概念向量
  │      · edit-plan tokens(连续)      = "被改区域目标长什么样" 给渲染器
  │
  ├─►【3】Mask 解码器 (LISA / SAM2 式)
  │      · query 向量 × 视觉骨干的逐帧稠密像素特征 → 逐像素分数
  │      · 时序传播(SAM2 式)→ 三层时空 mask
  │
  └─►【4】门控 Renderer (DiT, Wan2.2 级)
         条件 = 源latent + edit-plan tokens + 三层 mask
         核心机制:mask 门控的 inpainting
         · "不改变"区:每步去噪把 源latent 贴回(架构层强制钉死)
         · "直接/间接"区:在 tokens 引导下自由去噪
       ↓
     目标视频 V̂
```

## A.2 四块逐块说明

**【1】Abduction 源反演 —— 拿到不变量。**
一个 MLLM(Qwen2.5-VL 级)同时读源视频和指令。关键动作是把源视频**反演进 renderer 的 latent 空间**,得到 `源latent`。这个 latent 之后会一路喂给渲染器,作用是"凡是源视频里有、又不该被改的细节(身份、背景、无关物体与运动),都从这里原样取回"。这一步是后面"最小改动"能成立的根。

**【2】因果 Planner —— 决定改哪、改成啥。**
解析指令里的干预(`do(X=x)`),输出两类**向量**,注意都不是 mask:

- **区域 query 向量**:少量、稀疏的概念向量,代表"会被这次编辑波及的那些区块"。它不对齐像素,只是语义指针。
- **edit-plan tokens**:连续 token,描述被改区域应该变成什么样,作为渲染器的内容条件。

这两路全程可微,端到端训练,**不需要 RL**。

**【3】Mask 解码器 —— 把稀疏向量变成逐像素 mask。**
VLM 的 token 稀疏、不对齐像素,所以**不能让 planner 直接吐 mask**。这里接一个 LISA/SAM2 式解码器:用 query 向量去和视觉骨干输出的**逐帧稠密像素特征图**做匹配(点积/注意力),得到逐像素分数;再用 SAM2 式时序传播保证跨帧一致。产物是**三层时空 mask**(定义见 A.4)。

**【4】门控 Renderer —— mask 门控的 inpainting。**
一个强 DiT(Wan2.2 级)。它吃三路条件:① `源latent`(钉不变量/细节)、② `edit-plan tokens`(渲被改区)、③ 三层 mask(当时空门)。
**核心是门控机制**:每一步去噪,在"不改变"区把 `源latent` **贴回去**(不是靠模型自觉,是架构强制),只有"直接/间接"区在 tokens 引导下自由去噪。这样"最小改动"是**结构性保证**,不是优化出来的。
> 去掉 2nd-pass 后,接缝/morphing 必须在这一关解决:mask 边界做 feather + 门控区与保留区**联合去噪**(而非硬贴),避免接缝 artifact。

## A.3 mask 的两套来源(训练 vs 推理)—— 全系统命门
- **训练**:mask 是**精确**的。用仿真器(Kubric 式)以共享种子渲一对 factual/counterfactual 视频 + 物体级因果日志,直接抽出像素级精确 mask。双用途:(1) 门控渲染;(2) 当 GT 监督 planner 学画 mask。
- **推理(真实视频)**:**没有现成 mask,全靠 planner+解码器预测**,再去门控。
- **这就是最大风险**:漏预测 → 该变的被锁(漏后果);过预测 → 不该解锁的被解锁(过编)。"直接"区好办(分割成熟,甚至可让用户点选);**"间接"区(多跳因果连带)难,靠因果推理——模型上限基本被它卡住,是真正的创新点也是命门。**

## A.4 三层时空 mask(VOID quadmask 的通用化)
按"离干预点的因果距离"切,逐帧、非静态:

| 层 | 含义 | 对应 VOID |
|---|---|---|
| **直接** | 干预节点 + 一阶后代(作用点 + 1-hop) | remove ⊂ 此 |
| **间接** | 多跳传递后代(更远的因果连带;全局光照/阴影/反射归此) | affected(127) |
| **不改变** | 非后代 / 不变量 | preserve(255) |

落地细则:标签在**物体/属性级**给,mask 是其投影;像素冲突优先级 `直接 > 间接 > 不改变`;间接区**晚激活**(连带后果发生那一刻才亮),天然编码"先因后果"的时序。

## A.5 训练信号(描述性,不写公式)
- **主信号**:对仿真 GT 的反事实视频做 flow-matching / 去噪重建。
- **不变量保持**:"不改变"区生成出来的 latent 必须 match `源latent`(仿真共享种子时这监督最干净)。← Bernini 没有。
- **因果 mask 监督**:预测的三层 mask 要对齐仿真的物体级依赖图。
- **对齐(可选末层)**:人类偏好 / 可微 reward,做 RL 微调。核心训练不依赖它。

## A.6 数据与评测(一句话各一句)
- **训练数据**:仿真引擎 A(精确因果标签)为主 + t2v 增强 + 少量真实视频。
- **评测**:**人工标注的真实视频**为主集(只标"不变量集 I + 必现后果集 E",不需要整段 GT 视频);小份**异引擎仿真全 GT**做抽查校验。训练源与评测源严格不相交。
- **指标**:二维 **IP × CR**(不变量保持率 × 后果召回率),同时暴露过编与漏后果。

## A.7 现有最近邻(防撞车,知道每块谁做过)
- planner + mask 解码器 ≈ **X-Planner**(图像,2507.05259)
- mask 门控 + 源区域钉死渲染 ≈ **VACE**(Wan-14B,2503.07598)
- MLLM planner → 视频 DiT ≈ **Bernini** / **Omni-Video 2**(2602.08820)
- 因果推理驱动视频编辑 ≈ **Digital-Twin RL**(2511.14100)
- 因果区 mask + 仿真配对 + 扩散 ≈ **VOID**(2604.02296,删除特化)

**无人做、即本设计的真增量**:① abduction = 源 inversion 成 latent 当不变量先验;② 三层因果 mask 里的**间接/多跳**层;③ 绑定 abduction 的不变量保持损失。

# Part B — Pearl 叙事(用因果阶梯包装上面的系统)

> Part A 已经能独立读懂。这一部分只回答一个问题:**为什么这套工程恰好就是"做对反事实"的正确形状?** 用 Pearl 的语言把每一块对号入座。

## B.1 反事实 = 阶梯第三层(Rung 3)
Pearl 的因果阶梯:Rung 1 观察、Rung 2 干预、Rung 3 反事实。
- Rung 2 是**群体级**:`P(Y | do(x))`——"一般而言,推一把会怎样"。
- Rung 3 是**单元级**:"对**这一段**视频、这个具体场景,如果当时改了 X,会怎样"。
反事实编辑的本质就是 Rung 3:答案必须取决于**这一段源视频的具体状态**,而不只取决于指令。

## B.2 abduction → action → prediction,正好是我们的流水线
Pearl 给 Rung 3 的标准三步,和 Part A 的块**一一对应**:

| Pearl 步骤 | 含义 | 对应架构块 |
|---|---|---|
| **Abduction** | 用事实反推外生噪声 U 的后验(把"其余一切"固定下来) | 【1】源 inversion → `源latent` = 工程化的 U |
| **Action** | 施加干预 `do(X=x)` | 【2】Planner 解析指令出干预 |
| **Prediction** | 保持**同一个 U**,重算结果 | 【3】+【4】mask 门控:U 区贴回、后代区在干预下重渲 |

一句话:**`源latent` 就是被钉住的 U;"同一 U"在工程上就是"不改变区每步把源 latent 贴回去"。** 这正是 mask 门控机制的因果含义。

## B.3 为什么"最小改动 + 因果连带"是同一件事
- **不变量(非后代)**:从 abduction 重建并钉死 → 表现为"不改变"层 → 给出**最小改动**。
- **后果(后代)**:在场景结构上传播干预、按因果距离展开 → 表现为"直接/间接"层 → 给出**因果连带**。
属性类编辑 = 浅 DAG(直接层为主);物理力/事件 = 时序 rollout(间接层逐帧晚激活)。**同一机制,不同传播深度**——这就是统一两种语义的点。

## B.4 可识别性:为什么 benchmark 要分"可识别 / 欠定"
因果阶梯定理:Rung 3 一般**无法**由 Rung 1/2 唯一确定,除非加参数约束(单调性 / 双射生成机制)。
推论:**很多反事实的"正确答案"本身不唯一**。所以评测必须给每个 case 标 `可识别 / 欠定` flag:可识别例严判,欠定例只评"不变量"或"可接受集合",不强求唯一后果。

## B.5 Rung-3 闸门:benchmark 真的在测反事实的判据
> **同一指令 + 两段不同源视频 → 正确输出必须不同**(因为 abducted 出的 U 不同)。
若某模型的输出只由指令决定、与源无关,它做的只是 Rung 2 编辑,不是反事实——直接剔除。这条同时是**打 Bernini 的点**:Bernini 把反事实当 prompt 改写(prompt 主导),在"换源就该换答案"上会露馅。

## B.6 为什么这套在原理上赢 Bernini(三句话)
1. **门控源条件 = 结构性保不变量** → 治过编(Bernini 无不变量建模)。
2. **预测的因果连带 mask = 处理指令没说的后果** → 治漏后果(尤其间接/多跳)。
3. **abduction = 把生成锚回这一段源视频** → 过 Rung-3 的换源测试(Bernini prompt 主导,会挂)。
